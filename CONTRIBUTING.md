# Contributing to LYF SOS (Android)

Thank you for your interest in contributing to LYF SOS.

This project welcomes contributions **only if they respect its ethical,
privacy-first mission**.

---

## 🎯 Project Goals

LYF SOS exists to:
- Help users act during emergencies.
- Remain usable under stress.
- Avoid surveillance or misuse.
- Be honest about limitations.

Contributions must **strengthen these goals**, not dilute them.

---

## ✅ What We Welcome

- Bug fixes.
- Security improvements.
- Accessibility enhancements.
- Performance optimizations.
- Documentation improvements.
- Code readability improvements.

---

## ❌ What We Do NOT Accept

Pull requests that introduce:

- Background tracking.
- Analytics or telemetry.
- Advertising or monetization.
- AI-based behavior detection.
- Auto-triggered SOS.
- Hidden network calls.
- Dark patterns or panic UI.
- Feature bloat.

If a feature compromises privacy or consent, it will be rejected.

---

## 🔍 Contribution Rules

All contributions must:

1. Clearly explain **what** is changing.
2. Explain **why** it is necessary.
3. Describe **privacy impact**.
4. Describe **security impact**.
5. Respect offline-first behavior.

Unclear or rushed PRs will be closed.

---

## 🧪 Testing Expectations

- Code must compile.
- No new permissions without justification.
- No background services added.
- Offline behavior must remain predictable.

---

## 🛡️ Security Issues

Do **not** report security vulnerabilities publicly.

Follow `SECURITY.md` for responsible disclosure.

---

## 🧭 Code Style

- Keep logic simple.
- Prefer clarity over cleverness.
- Avoid unnecessary dependencies.
- Comment decisions that affect safety or privacy.

---

## ⚖️ License Agreement

By contributing, you agree that your contribution:
- Is your original work, or properly attributed.
- Is licensed under the project’s open-source license.

---

## 🤝 Final Note

LYF SOS is not a startup product.  
It is a **public safety tool**.

Contribute with restraint, responsibility, and empathy.
