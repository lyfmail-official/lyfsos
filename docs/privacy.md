# Privacy Policy — LYF SOS (Android)

LYF SOS is built on a single non-negotiable principle:

> **Your safety must not cost you your privacy.**

This document explains how LYF SOS handles user data, permissions, and device
access in a transparent and ethical manner.

---

## 🧭 Privacy Philosophy

LYF SOS follows these core rules:

- No accounts.
- No background tracking.
- No analytics.
- No ads.
- No third-party SDKs.
- No silent data collection.

If data is accessed, it is:
- Visible.
- Intentional.
- User-initiated.
- Temporary.

---

## 📦 Data LYF SOS Does NOT Collect

LYF SOS does **not** collect, store, or transmit:

- Names, phone numbers, or contacts.
- Email addresses.
- Device identifiers.
- Advertising IDs.
- Location history.
- Usage analytics.
- Crash analytics.
- Behavioral data.

There is **no central server** collecting user information.

---

## 📍 Location Data

### When Location Is Accessed
- Only when the user explicitly triggers SOS.
- Only while the SOS screen is active.

### How Location Is Used
- To display current position to the user.
- To allow the user to share it manually.

### What Is NOT Done
- No background location access.
- No continuous tracking.
- No location storage.
- No transmission without user action.

Location data lives **only in memory** and is discarded after use.

---

## 📶 Network Usage

LYF SOS:
- Does not auto-connect to servers.
- Does not sync data in background.
- Does not upload logs or telemetry.

Any network activity happens **only when the user chooses to share information**.

---

## 📡 Bluetooth & Nearby Features

If enabled by the user:

- Bluetooth is used only for nearby emergency signaling.
- No device scanning in background.
- No device fingerprinting.
- No persistent identifiers.

Bluetooth is optional and fully user-controlled.

---

## 📁 Local Storage

Stored locally (temporary):
- SOS draft message.
- User preferences.

Never stored:
- Location history.
- Emergency history.
- Identifiers.
- Logs.

All data is removed when the app is uninstalled.

---

## 🔐 Security Measures

- Minimal permission surface.
- OS-level encryption.
- No hardcoded keys.
- No remote command channels.

See `SECURITY.md` and `docs/threat-model.md`.

---

## 👤 Children & Vulnerable Users

LYF SOS:
- Does not target children.
- Does not profile users.
- Does not collect age or identity.

Use by minors should always involve parental or guardian awareness.

---

## 🔄 Changes to Privacy Policy

If privacy behavior changes:
- Documentation will be updated.
- Changes will be explicit and visible.
- No silent policy changes.
